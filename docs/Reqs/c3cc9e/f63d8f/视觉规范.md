# 视觉规范 — Setup database connection pool and base model

> 所属需求：后端 API 服务搭建

## 整体风格
**Modern Developer-Focused Aesthetic**: Clean, technical interface optimized for monitoring and debugging. Use monospace fonts for all technical data (URLs, timestamps, metrics) to enhance readability. Prioritize information density without clutter—employ subtle backgrounds and borders to create visual hierarchy. Status indicators use traffic-light color semantics (green=healthy, yellow=warning, red=error) with sufficient contrast for accessibility. Animations are minimal and purposeful (loading spinners, subtle hover states) to avoid distraction during critical monitoring tasks. Overall tone: professional, trustworthy, and immediately scannable for DevOps personnel.

## Health Check Status Badge

**Purpose**: Display database connection health status in monitoring dashboard

### States

#### Healthy State
- Background: `colors.status_healthy` with 10% opacity (#10B98119)
- Border: 1px solid `colors.status_healthy`
- Text: `colors.status_healthy`, font-size `typography.size_sm`, weight `typography.weight_medium`
- Icon: Check circle, 16px, `colors.status_healthy`
- Padding: `spacing.xs` `spacing.sm`
- Border-radius: `radius.full`
- Display format: "● Healthy" or "● healthy"

#### Unhealthy State
- Background: `colors.status_unhealthy` with 10% opacity (#EF444419)
- Border: 1px solid `colors.status_unhealthy`
- Text: `colors.status_unhealthy`, font-size `typography.size_sm`, weight `typography.weight_medium`
- Icon: X circle, 16px, `colors.status_unhealthy`
- Padding: `spacing.xs` `spacing.sm`
- Border-radius: `radius.full`
- Display format: "● Unhealthy"

#### Loading State
- Background: `colors.status_loading` with 10% opacity (#F59E0B19)
- Border: 1px solid `colors.status_loading`

- Icon: Spinner animation, 16px, `colors.status_loading`
- Padding: `spacing.xs` `spacing.sm`
- Border-radius: `radius.full`
- Display format: "◌ Connecting..."

- Border: 1px solid `colors.border`
- Border-radius: `radius.md`
- Padding: `spacing.md`
- Min-width: 120px

### Content Layout
- Label: "Response Time", `typography.size_xs`, `colors.text_secondary`, weight `typography.weight_normal`
- Value: "<number> ms", `typography.size_xl`, `colors.text_primary`, weight `typography.weight_semibold`, font-family `typography.font_mono`
- Spacing between label and value: `spacing.xs`

### Color Coding by Performance
- < 50ms: `colors.success`
- 50-100ms: `colors.status_loading`
- > 100ms: `colors.warning`

## Connection Pool Status Indicator

**Purpose**: Visualize connection pool usage (for admin dashboard)

### Visual Spec
- Container: Horizontal bar, height 8px, background `colors.bg_secondary`, border-radius `radius.full`
- Fill bar: Height 8px, background gradient from `colors.primary` to `colors.primary_hover`, border-radius `radius.full`
- Width: Percentage of pool_size usage (e.g., 5/10 = 50%)

### Labels
- Above bar: "Active Connections", `typography.size_sm`, `colors.text_secondary`
- Below bar: "5 / 10 (50%)", `typography.size_xs`, `colors.text_muted`, font-family `typography.font_mono`

### Warning States
- 70-90% usage: Fill bar changes to `colors.warning`
- > 90% usage: Fill bar changes to `colors.error`, add pulsing animation
