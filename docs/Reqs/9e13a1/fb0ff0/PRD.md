# PRD — Implement budget CRUD API endpoints

> 所属需求：预算管理功能

## 用户故事
As a 记账应用用户，I want to 设置和管理月度预算，So that 我可以控制支出并在超支前收到预警，避免过度消费。

## 功能需求
- 创建预算：用户可为总支出或特定分类设置月度预算金额
- 查询预算：用户可查看当前月份及历史月份的预算列表，支持按月份筛选
- 更新预算：用户可修改已设置的预算金额
- 删除预算：用户可删除不再需要的预算设置
- 预算进度计算：系统自动统计当前周期内的实际支出，计算预算使用百分比
- 权限隔离：用户只能操作自己创建的预算记录

## 验收标准
- [ ] POST /api/budgets 接口：amount 参数必须 > 0，period 格式必须为 YYYY-MM（如 2025-01），参数校验失败返回 400 状态码 + 具体错误信息
- [ ] POST /api/budgets 接口：成功创建后返回 201 状态码 + 完整预算对象（包含 id, user_id, category_id, amount, period, created_at）
- [ ] GET /api/budgets 接口：不传 period 参数时返回当前月份的所有预算，传入 period=2025-03 时仅返回该月份预算
- [ ] GET /api/budgets 接口：返回数据包含 used_amount（已使用金额）和 usage_percentage（使用百分比，保留 2 位小数）字段
- [ ] PUT /api/budgets/:id 接口：用户尝试修改他人预算时返回 403 状态码 + 错误提示「无权限操作此预算」
- [ ] DELETE /api/budgets/:id 接口：成功删除后返回 204 状态码，删除他人预算返回 403
- [ ] 预算进度计算逻辑：category_id 为 null 时统计该月所有支出，category_id 有值时仅统计该分类支出
- [ ] 预算进度计算逻辑：仅统计 type='expense' 的交易记录，income 类型不计入
- [ ] 所有接口响应时间 < 500ms（数据库查询 + 计算逻辑）
- [ ] 数据库 budgets 表包含字段：id (主键), user_id (外键), category_id (外键，可为 null), amount (decimal), period (string, 格式 YYYY-MM), created_at (timestamp)
- [ ] 数据库约束：同一用户同一月份同一分类（或总预算）只能有一条记录，违反时返回 409 状态码 + 提示「该月份预算已存在」

## 边界条件（不做的事）
- 不包含：预算超支自动锁定支出功能（仅提供数据，预警由前端实现）
- 不包含：预算模板功能（如自动复制上月预算）
- 不包含：多币种预算支持（当前仅支持单一货币）
- 暂不支持：周预算或年度预算（period 仅支持月度格式 YYYY-MM）
- 暂不支持：预算分享或多人协作预算
- 超出范围：预算建议算法（如基于历史数据推荐预算金额）
- 超出范围：预算执行报告生成（PDF/Excel 导出）
