# PRD — Build budget setting form and progress visualization UI

> 所属需求：预算管理功能

## 用户故事
As a 记账应用用户，I want to 设置和查看月度预算进度，So that 我可以实时掌握支出情况并避免超支。

## 功能需求
- 预算设置表单：支持选择预算类型（总预算/分类预算），输入金额（正整数或小数），选择月份（默认当前月）
- 预算类型切换：选择「总预算」时不显示分类选择器；选择「分类预算」时显示分类下拉框（复用现有支出分类数据）
- 预算进度展示：进度条显示「已用金额/预算总额」，百分比精确到整数位
- 超预算预警：进度 ≥ 100% 时进度条变红色，显示超支金额（如「已超支 ¥200」）
- 预算列表：展示当前月份所有预算（总预算置顶，分类预算按金额降序），每条显示：预算类型、金额、进度条、操作按钮（编辑/删除）
- 编辑功能：点击编辑按钮后表单回填现有数据，支持修改金额和月份（不可修改预算类型）
- 删除功能：点击删除按钮后弹出二次确认弹窗「确认删除该预算？」，确认后调用删除 API
- 超预算提示：当用户添加支出导致某预算超支时，前端显示 Toast「[分类名]预算已超支 ¥XX」（3 秒后自动消失）
- API 集成：调用后端 POST /budgets（创建）、GET /budgets?period=YYYY-MM（查询）、PUT /budgets/{id}（更新）、DELETE /budgets/{id}（删除）接口

## 验收标准
- [ ] 预算设置表单：切换预算类型后 100ms 内显示/隐藏分类选择器
- [ ] 金额输入框：只允许输入数字和小数点，最多 2 位小数，输入非法字符时实时拦截
- [ ] 月份选择器：默认值为当前月份（格式 YYYY-MM），可选范围为当前月份前后 12 个月
- [ ] 提交表单：点击保存按钮后 500ms 内调用 API，成功后显示 Toast「预算设置成功」并刷新列表
- [ ] 进度条：已用金额/预算总额 < 80% 时显示绿色，80%-99% 显示橙色，≥ 100% 显示红色
- [ ] 进度条文字：显示「¥已用/¥总额（百分比%）」，如「¥1,200/¥2,000（60%）」
- [ ] 超支提示：进度 ≥ 100% 时进度条下方显示红色文字「已超支 ¥XXX」
- [ ] 预算列表：总预算固定显示在第一行，分类预算按已用金额从高到低排序
- [ ] 空状态：当前月份无预算时显示空状态图 + 文案「暂无预算，点击上方按钮添加」
- [ ] 编辑功能：点击编辑按钮后 200ms 内表单回填数据，预算类型字段置灰不可修改
- [ ] 删除确认弹窗：点击删除按钮后 100ms 内弹出 Modal，标题「确认删除」，内容「删除后无法恢复，是否继续？」
- [ ] 删除成功：确认删除后 500ms 内调用 API，成功后显示 Toast「删除成功」并从列表移除该项
- [ ] 超预算 Toast：用户添加支出后，若导致某预算超支，300ms 内显示 Toast「[分类名]预算已超支 ¥XX」，3 秒后自动消失
- [ ] API 错误处理：网络请求失败时显示 Toast「操作失败，请重试」，超时时间设为 5s
- [ ] 响应式适配：移动端（<768px）表单字段纵向排列，进度条高度 8px；桌面端（≥1024px）表单字段横向排列，进度条高度 12px
- [ ] 加载状态：查询预算列表时显示骨架屏（3 行占位块），加载完成后 200ms 内渲染真实数据

## 边界条件（不做的事）
- 不包含：预算周期自定义（仅支持按月），预算模板功能，预算分享功能
- 暂不支持：多币种预算，预算自动结转到下月，预算与账户关联（仅按用户维度）
- 超出范围：预算分析报表（如趋势图、同比环比），预算建议算法，预算通知推送（邮件/短信）
- [假设] 预算金额范围：0.01 - 9,999,999.99（前端校验 + 后端校验）
- [假设] 同一用户同一月份同一分类只能设置一个预算（后端唯一索引约束）
- [假设] 删除预算不影响历史支出记录，仅停止进度统计
- [假设] 预算进度实时计算（每次查询时聚合当月该分类的支出总额）

## 资产需求线索
- 图标：预算设置图标（钱袋子/目标靶心），编辑图标（铅笔），删除图标（垃圾桶），预警图标（感叹号三角形）
- 空状态插画：空钱包或空列表插画（尺寸 200x200px，SVG 格式）
- 进度条颜色：绿色 #52c41a，橙色 #faad14，红色 #ff4d4f（复用 Ant Design 色板）
- Toast 样式：成功 - 绿色背景 + 白色文字，失败 - 红色背景 + 白色文字，警告 - 橙色背景 + 白色文字
- 骨架屏：灰色占位块（#f0f0f0），带呼吸动画（1.5s 循环）
- 二次确认弹窗：使用 Ant Design Modal 组件默认样式，确认按钮为红色危险按钮
