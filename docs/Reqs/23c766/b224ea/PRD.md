# PRD — E2E: Test data backup and restore workflow

> 所属需求：数据备份与恢复

## 用户故事
As a 项目管理员，I want to 通过端到端测试验证数据备份与恢复的完整流程，So that 确保用户数据安全功能在生产环境中可靠运行，避免数据丢失风险。

## 功能需求
- 测试数据生成：通过多次访问页面生成至少 10 条测试数据记录
- 数据导出验证：点击导出按钮后下载 JSON 文件，验证文件格式符合 JSON 规范且包含所有生成的记录
- 数据清空验证：点击清空按钮后弹出二次确认对话框，确认后数据被清空且页面计数归零
- 数据导入验证：上传之前导出的 JSON 文件，验证数据恢复成功且计数与导出前一致
- 异常场景测试：上传非法 JSON 文件时显示错误提示，取消清空确认时数据保持不变
- 边界场景测试：空数据状态下导出 JSON 文件，重复导入同一文件时数据不重复
- 截图记录：每个关键步骤自动截图保存到 test-results/ 目录

## 验收标准
- [ ] 测试脚本运行时间 ≤ 60s（包含所有步骤）
- [ ] 生成测试数据后，页面显示的访问计数 ≥ 10
- [ ] 点击导出按钮后 2s 内触发浏览器下载，文件名格式为 `visit_data_YYYYMMDD_HHmmss.json`
- [ ] 导出的 JSON 文件通过 `JSON.parse()` 验证无语法错误
- [ ] 导出的 JSON 文件包含 `visits` 数组，数组长度等于页面显示的计数
- [ ] 点击清空按钮后 500ms 内弹出确认对话框，对话框包含「确定」和「取消」按钮
- [ ] 确认清空后 1s 内页面计数显示为 0，再次导出的 JSON 文件 `visits` 数组长度为 0
- [ ] 上传之前导出的 JSON 文件后 2s 内页面计数恢复到导出前的数值
- [ ] 导入后再次导出的 JSON 文件内容与首次导出的文件内容一致（timestamp 可忽略微小差异 ≤ 1s）
- [ ] 上传非法 JSON 文件（如纯文本 `invalid data`）时 1s 内显示错误提示 Toast，文案包含「格式错误」或「JSON」关键词
- [ ] 点击清空按钮后点击「取消」，页面计数保持不变
- [ ] 空数据状态（计数为 0）下点击导出按钮，下载的 JSON 文件 `visits` 数组长度为 0 且文件大小 < 1KB
- [ ] 重复导入同一 JSON 文件 2 次，页面计数不翻倍（幂等性验证）
- [ ] 测试过程中至少生成 5 张截图，保存路径为 `test-results/backup-restore-*.png`
- [ ] 测试脚本使用 Playwright 的 `expect()` 断言，所有断言通过时测试标记为 PASS

## 边界条件（不做的事）
- 不包含：多用户并发导入导出的竞态测试
- 不包含：超大数据量（> 10000 条记录）的性能测试
- 不包含：跨浏览器兼容性测试（仅在 Chromium 内核运行）
- 不包含：数据加密/解密功能的测试
- 暂不支持：导出为 CSV 或其他格式的测试
- 暂不支持：增量备份/差异备份的测试
- 超出范围：后端 API 的单元测试（由其他工单覆盖）
- 超出范围：数据库直接操作的验证（仅测试前端交互）
