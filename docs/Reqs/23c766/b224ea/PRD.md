# PRD — E2E: Test data backup and restore workflow

> 所属需求：数据备份与恢复

## 用户故事
As a 项目管理员，I want to 通过 E2E 测试验证数据备份与恢复的完整流程，So that 确保用户数据导出、清空、导入功能在真实浏览器环境下正常工作且数据完整性得到保障。

## 功能需求
- 测试数据生成：通过多次访问页面生成至少 10 条测试数据记录
- 导出功能测试：点击导出按钮，验证浏览器下载 JSON 文件，文件格式符合规范且包含所有记录
- 清空功能测试：点击清空按钮，验证二次确认对话框出现，确认后数据清空且计数归零
- 导入功能测试：上传之前导出的 JSON 文件，验证数据恢复成功且计数与导出前一致
- 异常场景测试：上传非法 JSON 文件时显示错误提示，取消清空确认时数据不变
- 边界场景测试：空数据状态下导出空 JSON 数组，重复导入同一文件数据不重复
- 截图记录：每个关键步骤自动截图保存到 test-results/ 目录

## 验收标准
- [ ] 生成测试数据后，页面计数显示 ≥ 10 条记录
- [ ] 点击导出按钮后 2s 内浏览器下载文件，文件名格式为 `backup_YYYYMMDD_HHmmss.json`
- [ ] 导出的 JSON 文件包含 `version`、`timestamp`、`data` 字段，`data` 数组长度等于页面显示的记录数
- [ ] 点击清空按钮后 500ms 内弹出确认对话框，对话框包含「确定」和「取消」按钮
- [ ] 点击确认后 1s 内数据清空，页面计数显示为 0，列表显示空状态提示
- [ ] 上传有效 JSON 文件后 2s 内数据恢复完成，页面计数与导出前一致，列表内容完全匹配
- [ ] 上传非 JSON 文件或格式错误的 JSON 时，3s 内显示错误提示「文件格式错误，请上传有效的备份文件」
- [ ] 点击清空对话框的「取消」按钮后，对话框关闭且数据不变，计数保持不变
- [ ] 空数据状态下导出文件，JSON 文件 `data` 字段为空数组 `[]`
- [ ] 重复导入同一文件 2 次，数据不重复，计数保持与文件中记录数一致
- [ ] 每个测试步骤（生成数据、导出、清空、导入）自动截图，截图文件保存到 `test-results/screenshots/` 目录
- [ ] 测试运行时间 ≤ 60s（不含浏览器启动时间）
- [ ] 测试失败时生成详细错误日志，包含失败步骤、期望值、实际值和截图路径

## 边界条件（不做的事）
- 不包含：性能压力测试（如导出 10 万条数据）
- 不包含：多浏览器兼容性测试（仅测试 Chromium）
- 不包含：并发导入导出测试（单用户顺序操作）
- 不包含：数据加密/解密功能测试
- 不包含：云端备份同步功能
- 暂不支持：移动端触摸操作测试
- 超出范围：备份文件版本兼容性测试（仅测试当前版本格式）
- 超出范围：网络异常场景模拟（如上传中断）
