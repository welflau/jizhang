# PRD — Frontend: Add data backup and restore UI

> 所属需求：数据备份与恢复

## 用户故事
As a 系统管理员，I want to 导出、导入和清空访问数据，So that 可以备份重要数据、迁移历史记录或重置系统状态。

## 功能需求
- 数据导出：点击「导出数据」按钮，调用 GET /api/export，触发浏览器下载 JSON 文件，文件名格式为 visits_backup_YYYYMMDD_HHMMSS.json（如 visits_backup_20260501_143022.json）
- 数据导入：提供文件上传控件（accept=".json"），选择 JSON 文件后调用 POST /api/import，上传完成后显示导入结果（成功导入 X 条，失败 Y 条）
- 数据清空：点击「清空数据」按钮，弹出确认对话框，标题「确认清空」，内容「确定要清空所有访问数据吗？此操作不可恢复！」，提供「取消」和「确认清空」两个按钮，点击「确认清空」后调用 DELETE /api/clear
- 操作状态反馈：导入/清空操作期间显示 loading 状态（按钮禁用 + Spin 组件），操作完成后刷新页面访问计数显示
- 错误处理：API 调用失败时通过 message.error() 显示具体错误信息（如「导入失败：文件格式错误」「清空失败：服务器错误」）

## 验收标准
- [ ] 点击「导出数据」按钮后 500ms 内触发浏览器下载，文件名符合 visits_backup_YYYYMMDD_HHMMSS.json 格式，文件内容为有效 JSON 数组
- [ ] 文件上传控件仅接受 .json 文件，选择非 JSON 文件时浏览器原生拦截或前端提示「请选择 JSON 文件」
- [ ] 上传 JSON 文件后 3s 内调用 /api/import 并显示结果 Toast，格式为「导入成功：X 条记录」或「导入失败：Y 条记录失败」
- [ ] 点击「清空数据」按钮后 200ms 内弹出确认对话框，对话框包含标题「确认清空」、警告文案「确定要清空所有访问数据吗？此操作不可恢复！」、「取消」和「确认清空」两个按钮
- [ ] 点击对话框「取消」按钮后对话框关闭，不调用 API
- [ ] 点击对话框「确认清空」按钮后 500ms 内调用 DELETE /api/clear，成功后显示「数据已清空」Toast 并刷新访问计数为 0
- [ ] 导入/清空操作期间，对应按钮显示 loading 状态（禁用 + Spin 图标），操作完成后恢复可点击状态
- [ ] 导入/清空操作完成后 1s 内刷新页面访问计数显示（调用现有的访问统计 API）
- [ ] API 调用失败时显示 message.error()，错误文案包含具体原因（如「网络错误」「文件格式错误」「服务器错误」），持续 3s 后自动消失
- [ ] 数据管理功能区位于页面底部或独立 Tab，与访问统计图表区域视觉分离（间距 ≥ 24px）
- [ ] 导出按钮使用 DownloadOutlined 图标，导入区域使用 UploadOutlined 图标，清空按钮使用 DeleteOutlined 图标并标记为 danger 类型（红色）

## 边界条件（不做的事）
- 不包含：数据导出格式选择（仅支持 JSON，不支持 CSV/Excel）
- 不包含：增量导入（导入时全量覆盖，不支持合并或去重）
- 不包含：导入前数据预览（选择文件后直接上传，不展示文件内容）
- 不包含：定时自动备份功能（仅支持手动导出）
- 不包含：导入进度条（文件上传使用一次性请求，不显示百分比进度）
- 不包含：清空操作的撤销功能（清空后数据永久删除，无法恢复）
- 暂不支持：批量文件导入（一次仅支持上传一个 JSON 文件）
- 暂不支持：云端备份（导出文件保存到本地，不上传到云存储）

## 资产需求线索
- 图标：DownloadOutlined（导出按钮）、UploadOutlined（导入区域）、DeleteOutlined（清空按钮）、ExclamationCircleOutlined（确认对话框警告图标）
- 插画/图片：暂无
- 音效：暂无
- 动画：Spin 组件（loading 状态）、Modal 弹出动画（Ant Design 默认）、message Toast 动画（Ant Design 默认）
