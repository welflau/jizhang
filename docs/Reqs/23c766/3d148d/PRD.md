# PRD — Frontend: Add data backup and restore UI

> 所属需求：数据备份与恢复

## 用户故事
As a 系统管理员，I want to 导出、导入和清空访问记录数据，So that 可以备份数据防止丢失、迁移数据到新环境、或重置系统进行测试。

## 功能需求
- 数据导出：点击「导出数据」按钮，调用 GET /api/export，浏览器自动下载 JSON 文件，文件名格式为 visits_backup_YYYYMMDD_HHMMSS.json
- 数据导入：通过文件上传控件选择 JSON 文件，调用 POST /api/import，上传完成后显示导入结果（成功 X 条，失败 Y 条）
- 数据清空：点击「清空数据」按钮，弹出二次确认对话框，确认后调用 DELETE /api/clear，清空所有访问记录
- 状态反馈：导入/清空操作期间显示 loading 状态，操作完成后刷新页面访问计数显示
- 错误处理：API 调用失败时显示友好错误提示（如：文件格式错误、网络异常等）

## 验收标准
- [ ] 点击「导出数据」按钮后 500ms 内触发浏览器下载，文件名格式为 visits_backup_20260501_143022.json（年月日_时分秒）
- [ ] 导出的 JSON 文件包含所有访问记录字段：id, ip, user_agent, referer, path, timestamp, country, region, city
- [ ] 文件上传控件仅接受 .json 扩展名文件，选择非 JSON 文件时显示提示「请选择 JSON 格式文件」
- [ ] 导入完成后 1s 内显示结果 Toast：「导入成功 X 条，失败 Y 条」，持续 3s 后消失
- [ ] 点击「清空数据」按钮后 200ms 内弹出确认对话框，标题为「确认清空数据」，内容为「确定要清空所有数据吗？此操作不可恢复！」，包含「取消」和「确定」按钮
- [ ] 二次确认对话框中点击「取消」后对话框关闭，不执行任何操作
- [ ] 二次确认对话框中点击「确定」后 300ms 内显示 loading 状态（按钮文字变为「清空中...」且禁用）
- [ ] 清空操作成功后 1s 内显示 Toast「数据已清空」，持续 2s 后消失，同时页面访问计数刷新为 0
- [ ] 导入/清空操作期间，对应按钮显示 loading 图标并禁用，防止重复提交
- [ ] API 返回 4xx/5xx 错误时，显示错误 Toast：「操作失败：[错误信息]」，持续 5s 后消失
- [ ] 导入文件格式错误（非 JSON 或缺少必需字段）时，显示 Toast「文件格式错误，请检查后重试」
- [ ] 网络请求超时（> 30s）时显示 Toast「请求超时，请检查网络连接」
- [ ] 导入/清空操作完成后，页面顶部访问计数在 500ms 内自动刷新为最新值
- [ ] 数据管理功能区位于页面底部，包含三个操作按钮：「导出数据」「导入数据」「清空数据」，按钮间距 16px
- [ ] 移动端（屏幕宽度 < 768px）时，三个按钮垂直排列，每个按钮宽度 100%

## 边界条件（不做的事）
- 不包含：数据加密功能（导出的 JSON 为明文）
- 不包含：增量导入（每次导入会覆盖现有数据或追加，具体行为由后端 API 决定）
- 不包含：导入前数据校验预览（直接上传执行）
- 不包含：导出数据筛选功能（导出全量数据）
- 不包含：定时自动备份功能
- 暂不支持：导出为 CSV/Excel 等其他格式
- 暂不支持：批量文件上传（一次仅支持上传一个 JSON 文件）
- 超出范围：数据版本管理和回滚功能

## 资产需求线索
- 图标：导出图标（download icon）、导入图标（upload icon）、清空图标（delete/trash icon）
- 图标：loading 状态图标（spinner）
- 图标：确认对话框警告图标（warning icon）
- 插画：暂无
- 音效：暂无
- 动画：按钮点击时的 loading 旋转动画（使用 Ant Design 内置 Spin 组件）
