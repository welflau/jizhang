# 视觉规范 — Implement responsive layout breakpoints and grid system

> 所属需求：响应式布局适配

## 整体风格
采用现代扁平化设计风格，遵循 Ant Design 设计语言规范。布局系统强调流畅的响应式过渡动画（300ms ease），确保在不同断点切换时视觉连贯性。色彩方案支持浅色/深色双主题，优先使用系统字体栈保证跨平台一致性。栅格系统采用灵活的列数配置（桌面 24 列、平板 12 列、移动 6 列），间距遵循 8px 基准网格系统。所有交互元素保持 4.5:1 以上的对比度以满足 WCAG AA 无障碍标准。

## 组件视觉规范

# 响应式布局组件视觉规范

## 1. ResponsiveContainer（响应式容器）

### 桌面端（≥1024px）
- **最大宽度**：1440px
- **水平对齐**：居中（margin: 0 auto）
- **左右内边距**：24px
- **背景色**：透明（继承父级）
- **过渡动画**：padding 300ms ease

### 平板端（768-1023px）
- **宽度**：100%
- **左右内边距**：16px
- **背景色**：透明
- **过渡动画**：padding 300ms ease

### 移动端（<768px）
- **宽度**：100%
- **左右内边距**：12px
- **背景色**：透明
- **过渡动画**：padding 300ms ease

### 通用属性
- **盒模型**：border-box
- **溢出处理**：overflow-x: hidden
- **最小高度**：无限制

---

## 2. Grid System（栅格系统）

### Row 组件
- **显示方式**：flex
- **换行**：wrap
- **对齐方式**：flex-start（可配置）
- **间距（gutter）**：
  - xs（<576px）：8px
  - sm（576-767px）：12px
  - md（768-1023px）：16px
  - lg（≥1024px）：24px
- **负边距**：根据 gutter 自动计算（-gutter/2）
- **过渡动画**：margin 300ms ease

### Col 组件
- **显示方式**：block
- **盒模型**：border-box
- **内边距**：根据 Row 的 gutter 自动计算（gutter/2）
- **列宽计算**：
  - 桌面端（24列）：width = (span / 24) * 100%
  - 平板端（12列）：width = (span / 12) * 100%
  - 移动端（6列）：width = (span / 6) * 100%
- **过渡动画**：width 300ms ease, padding 300ms ease

---

## 3. 响应式间距工具类

### .p-responsive（响应式内边距）
- **桌面端**：padding: 24px
- **平板端**：padding: 16px
- **移动端**：padding: 12px
- **过渡动画**：padding 300ms ease

### .m-responsive（响应式外边距）
- **桌面端**：margin: 24px
- **平板端**：margin: 16px
- **移动端**：margin: 12px
- **过渡动画**：margin 300ms ease

### .gap-responsive（响应式间隙）
- **桌面端**：gap: 24px
- **平板端**：gap: 16px
- **移动端**：gap: 12px
- **过渡动画**：gap 300ms ease

---

## 4. 断点指示器（开发调试用）

### 视觉样式
- **位置**：固定在右下角（bottom: 16px, right: 16px）
- **尺寸**：80px × 32px
- **背景色**：rgba(0, 0, 0, 0.75)
- **文字颜色**：#ffffff
- **字体大小**：12px
- **字体粗细**：500
- **圆角**：4px
- **内边距**：8px 12px
- **层级**：z-index: 9999
- **显示内容**：

  - 平板端："Tablet 768-1023px"
  - 移动端："Mobile <768px"
- **生产环境**：display: none

## 5. 媒体查询断点

### CSS 变量定义
```css
:root {
  --breakpoint-mobile: 768px;
  --breakpoint-tablet: 1024px;
  --breakpoint-desktop: 1440px;
}
```

### 媒体查询规则
- **mobile-only**：@media (max-width: 767px)
- **tablet-only**：@media (min-width: 768px) and (max-width: 1023px)
- **tablet-and-up**：@media (min-width: 768px)
- **desktop-only**：@media (min-width: 1024px)
- **desktop-large**：@media (min-width: 1440px)

- **边框**：无
- **阴影**：无
- **背景色**：透明

### 调试模式（开发环境）
- **边框**：1px dashed #d9d9d9
- **背景色**：rgba(24, 144, 255, 0.02)
- **提示文字**：显示当前断点和容器宽度

## 7. 滚动条样式（桌面端）

### 滚动条轨道
- **宽度**：8px
- **背景色**：#f5f5f5
- **圆角**：4px

### 滚动条滑块
- **背景色**：#d9d9d9
- **圆角**：4px
- **悬停背景色**：#bfbfbf
- **过渡动画**：background-color 150ms ease

### 移动端
- **滚动条**：使用系统默认样式（不自定义）

---

## 8. 性能优化

### 过渡动画
- **仅应用于**：padding、margin、width、gap
- **避免动画**：height（使用 max-height 替代）、position
- **硬件加速**：transform 和 opacity 优先

### 重绘优化
- **使用 will-change**：仅在必要时应用（padding、width）
- **移除时机**：动画结束后移除 will-change

---

## 9. 无障碍支持

### 焦点可见性
- **断点切换时**：保持当前焦点元素可见
- **键盘导航**：确保 Tab 顺序在布局变化后仍然合理

### 屏幕阅读器
- **容器角色**：role="main" 或 role="region"
- **断点变化通知**：使用 aria-live="polite" 区域通知布局变化（可选）

---

## 10. 颜色与对比度

### 主题适配
- **浅色主题**：
  - 背景色：#ffffff
  - 边框色：#d9d9d9
  - 文字色：#262626
- **深色主题**：
  - 背景色：#141414
  - 边框色：#434343
  - 文字色：#ffffff

### 对比度要求
- **文字与背景**：≥4.5:1（WCAG AA 标准）
- **边框与背景**：≥3:1
