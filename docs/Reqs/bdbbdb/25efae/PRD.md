# PRD — Design category icon set and color palette

> 所属需求：分类管理系统

## 用户故事
As a 记账应用用户，I want to 为收支记录选择合适的图标和颜色，So that 能快速识别不同分类，提升记账体验的视觉友好度和操作效率。

## 功能需求
- 图标集合：设计至少 20 个常用分类图标（餐饮、交通、购物、医疗、娱乐、工资、奖金、投资等），SVG 格式，支持单色填充
- 配色方案：提供 12-16 种预设颜色，符合 WCAG 2.1 AA 级对比度标准（文字对比度 ≥ 4.5:1，图标对比度 ≥ 3:1）
- 图标规格：统一 24x24px 视口，笔画宽度 1.5-2px，圆角 2px，保持视觉一致性
- 颜色分组：收入类使用暖色系（绿/蓝），支出类使用冷/中性色系（红/橙/灰），默认分类使用品牌色
- 导出格式：SVG 源文件 + React 组件封装（支持 props 传入 color/size）
- 文档输出：图标预览表（名称+图标+推荐场景）+ 配色表（色值+对比度数据+使用建议）

## 验收标准
- [ ] 图标数量 ≥ 20 个，覆盖常见收支分类（餐饮、交通、购物、医疗、娱乐、住房、通讯、工资、奖金、投资、礼金等至少 15 个场景）
- [ ] 所有图标 SVG 文件大小 ≤ 2KB，viewBox="0 0 24 24"，无内嵌样式
- [ ] 配色方案包含 12-16 种颜色，每种颜色在白色背景上对比度 ≥ 4.5:1（使用 WebAIM Contrast Checker 验证）
- [ ] 提供 Figma/Sketch 源文件 + 导出的 SVG 文件夹（按图标名称命名，如 food.svg, transport.svg）
- [ ] React 组件支持 `<CategoryIcon name="food" color="#FF6B6B" size={24} />` 调用方式
- [ ] 输出设计文档（Markdown 格式）包含：图标预览表（3 列布局：图标+名称+推荐分类）+ 配色表（色值+Hex/RGB+对比度数值+使用场景）
- [ ] 图标在 16px/24px/32px 三种尺寸下清晰可辨，无模糊或变形
- [ ] 颜色命名遵循语义化（如 income-primary, expense-food, neutral-default），不使用 color1/color2
- [ ] 设计文档中标注无障碍使用建议（如色盲友好组合、必须配合文字标签的场景）

## 边界条件（不做的事）
- 不包含：动画效果、渐变色、多色图标（仅支持单色填充）
- 不包含：3D 图标、写实风格插画、品牌 Logo（如支付宝/微信图标）
- 暂不支持：用户自定义上传图标（仅提供预设图标集）
- 暂不支持：深色模式适配（Phase 1 仅适配浅色背景）
- 超出范围：图标动效库、Lottie 动画、骨骼动画

## 资产需求线索
- 图标：20+ 个 SVG 图标文件（food.svg, transport.svg, shopping.svg, salary.svg 等），统一风格线性图标
- 设计规范文档：《分类图标设计规范.md》，包含网格系统、笔画规范、圆角标准
- 配色表文档：《配色方案与无障碍标准.md》，包含 Hex 色值、对比度测试结果、使用示例
- React 组件代码：`CategoryIcon.jsx` + `iconMap.js`（图标名称到 SVG 路径的映射）
- Figma/Sketch 源文件：包含所有图标的可编辑版本 + 颜色样式库
- 预览页面：HTML 静态页面展示所有图标和颜色组合效果（可选，方便评审）
